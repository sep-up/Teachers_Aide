1. 引言
1.1 项目背景
教师在日常工作中需要处理大量的教务事务，包括成绩录入、学生评语撰写、考勤统计、课程表管理等。现有通用工具往往功能繁杂，不够聚焦，或者操作复杂。本项目命名为 Teachers Aide，旨在为一线教师提供轻量、高效、聚焦核心场景的工作助手，帮助教师提升工作效率，减少重复劳动。

1.2 项目目标
开发一款微信小程序 Teachers Aide，为中小学教师提供以下核心功能：

班级与学生信息管理

考试成绩录入与智能分析

学生评语智能生成与自定义模板

简易课程表管理

学生考勤记录

班级通知快速发布与分享

1.3 目标用户
中小学班主任、任课教师。

1.4 术语表
术语	解释
教师端	教师使用的小程序界面
班级	教师所管理的行政班或教学班
评语模板	预定义或自定义的评语片段，可组合生成完整评语
订阅消息	微信小程序能力，用于向用户发送服务通知
2. 总体描述
2.1 产品范围
核心功能：班级管理、成绩管理、评语生成、课程表、考勤、通知。

边界：小程序仅供教师使用，不涉及家长登录；通知通过生成图片分享至微信群实现，暂不开发独立家长端。

2.2 用户场景
场景一：期中考试后，张老师需要快速录入全班50人的成绩，并查看班级平均分、各分数段人数，以便分析教学效果。

场景二：期末写评语时，李老师面对几十名学生，希望能根据学生成绩和平时表现自动生成评语草稿，再手动微调，提高效率。

场景三：王班主任想查看今天谁没来上课，并记录缺勤原因。

场景四：赵老师调课后，需要快速查看更新后的课表，并提醒自己下一节课的时间。

场景五：刘老师需要发布周末家长会通知，希望快速生成一张精美的通知图片发到家长群。

2.3 假设与约束
开发环境：微信小程序原生开发（JavaScript / WXML）或使用 UniApp / Taro 等跨端框架；后端使用微信云开发（Node.js + 云数据库）。

数据存储：使用微信云开发数据库，文件（如学生头像、通知图片）使用云存储。

用户认证：通过微信小程序提供的 wx.login 获取 openid 作为用户唯一标识，无需注册手机号。

支付功能：初期不考虑付费功能，后期可通过广告（流量主）获得收益。

隐私合规：需在《用户隐私协议》中明确说明收集的学生信息仅用于本地数据处理，不上传至任何第三方服务器。

3.2 功能详情
3.2.1 用户登录与授权
功能描述：教师首次进入小程序需授权微信登录，后续自动登录。

输入/前置条件：无

处理流程：

调用 wx.login 获取临时 code，传给云函数换取 openid。
查询数据库 users 表是否存在该 openid，若不存在则创建新用户（默认昵称等）。
将用户信息存入全局状态。
输出/后置条件：进入首页（班级列表/工作台）。

异常处理：网络失败时提示重试。

3.2.2 班级管理
功能描述：教师可创建多个班级，编辑班级信息，切换当前班级。

功能点：

创建班级：输入班级名称（如“三年级2班”）、学年、年级、学科（可选）。班级唯一标识由系统生成。

班级列表：以卡片形式展示所有班级，支持左右滑动或下拉切换。

编辑班级：修改班级信息。

删除班级：需二次确认，删除后班级内所有数据（学生、成绩、考勤等）一并删除（逻辑删除或物理删除）。

数据字段：

班级ID (classId)

班级名称

学年 (如 "2025-2026")

年级 (如 "三年级")

学科 (任课教师选填，如 "数学")

创建时间

所属教师 openid

3.2.3 学生管理
功能描述：管理班级内的学生名单。

功能点：

添加学生：支持单个添加（姓名、性别、学号、家长联系方式可选）和批量导入（通过 Excel 模板导入，模板可下载）。

学生列表：按学号或姓名排序，显示学生姓名、性别、学号。

编辑学生：修改学生信息。

删除学生：二次确认，删除后该学生的所有成绩、考勤记录一并删除。

学生详情：可查看该学生的个人档案（包括所有成绩、出勤记录等）。

数据字段：

学生ID (studentId)

姓名

性别

学号 (可选)

家长手机号 (可选)

所属班级ID

入班日期

状态 (在读/毕业)

3.2.4 成绩管理
功能描述：录入考试成绩，自动生成统计分析。

功能点：

考试管理：创建考试，输入考试名称（如“期中考试”）、考试日期、科目。一个班级下可有多场考试。

成绩录入：

列表模式：展示所有学生，逐行输入分数（支持键盘上下切换快速录入）。

支持缺考标记（输入“/”或特殊符号）。

支持分数校验（如0-100整数或小数，保留一位小数）。

成绩统计：自动计算全班平均分、最高分、最低分、优秀率（≥90）、及格率（≥60）、各分数段人数分布（0-59,60-69,70-79,80-89,90-100）。

成绩查看：按考试查看成绩列表，支持按分数高低排序。

成绩导出：将当前考试的成绩单导出为图片（分享用）或Excel文件（保存用）。图片格式包含班级、考试名称、学生姓名、分数。

数据字段：

成绩ID

学生ID

考试ID

分数 (浮点数)

缺考标记 (布尔)

备注

3.2.5 评语生成
功能描述：辅助教师撰写期末或平时评语，提高效率。

功能点：

评语模板库：

系统预置常见评语模板（如“学习认真”、“积极发言”、“团结同学”等），按维度分类（学习态度、纪律、品德、特长）。

教师可自定义模板，新增、编辑、删除自己的模板。

评语生成器：

选择班级，显示学生列表。

点击学生进入评语编辑页，页面包含：

该生成绩概览（最近一次考试各科成绩，可选）

评语模板选择区（可多选，组合生成）

生成评语草稿（将选中的模板片段拼接，自动添加连接词）

手动编辑框（教师可自由修改）

支持保存评语到该生档案。

评语导出：可导出全班学生评语为Excel（含姓名、评语），或生成一张张评语卡片图片（用于私发家长）。

数据字段：

评语ID

学生ID

评语内容

学期标识 (如“2025下”)

生成时间

模板ID列表 (可选)

3.2.6 课程表管理
功能描述：简单课表查看与编辑。

功能点：

课程表显示：按周视图展示（周一至周日，每天节数可由教师自定义，如上午4节、下午3节）。每个格子显示课程名称、地点（可选）。

编辑模式：

添加课程：选择星期、节次，输入课程名称、地点。

编辑/删除课程。

支持调课：长按某节课拖拽到另一位置，或点击调课按钮交换两节课。

提醒设置：可设置课前提醒（默认提前10分钟），通过微信订阅消息发送。

数据字段：

课表ID

班级ID

星期几 (1-7)

节次 (如第1节)

课程名称

地点

教师姓名 (可选)

颜色标记

3.2.7 考勤管理
功能描述：记录学生每日出勤情况。

功能点：

考勤记录：选择日期（默认当天），展示班级学生列表，可标记出勤状态（正常、迟到、早退、请假、缺勤）。支持全勤一键标记。

考勤统计：查看某时间段内（周/月）每个学生的出勤统计（迟到次数、请假次数、缺勤次数）。

考勤导出：导出考勤统计表。

数据字段：

考勤ID

学生ID

日期

状态 (枚举：0-正常，1-迟到，2-早退，3-请假，4-缺勤)

备注 (如请假原因)

3.2.8 通知公告
功能描述：快速生成通知图片并保存到相册，供教师分享到微信群。

功能点：

编辑通知：输入标题、正文、落款（教师姓名/班级）、日期。

模板选择：提供几种简洁美观的卡片样式供选择（字体、背景色、边框）。

生成图片：将编辑好的通知渲染成图片（可使用 canvas 或第三方绘图库），预览后保存到手机相册。

历史通知：保存已生成的通知图片到云存储，可在小程序内查看历史记录。

数据字段：

通知ID

标题

正文

落款

生成时间

图片URL (云存储)

3.2.9 个人中心
功能描述：教师个人信息设置、数据备份与恢复、帮助反馈。

功能点：

个人信息：显示微信昵称、头像。

数据备份与恢复：支持将全部班级数据备份到云存储（导出为JSON文件），并可从备份文件恢复。

帮助与反馈：常见问题列表、联系开发者入口。

关于我们：版本号、隐私协议、用户协议。

4. 非功能需求
4.1 性能要求
页面首次加载时间不超过2秒（在良好网络环境下）。

学生列表滚动流畅，单页加载50条数据无卡顿。

云数据库查询响应时间不超过1秒。

4.2 易用性要求
UI 设计简洁清晰，符合微信小程序设计规范。

关键操作（如删除、批量导入）有二次确认。

提供新手引导，首次进入班级管理时提示如何创建班级。

4.3 可靠性
云数据库定期备份（由微信云开发自动完成）。

用户数据隔离：不同教师只能访问自己创建的数据。

批量导入数据时支持事务，部分失败时回滚并提示错误行。

4.4 兼容性
支持 iOS 微信 7.0+ 和 Android 微信 7.0+。

适配不同屏幕尺寸（包括平板）。

4.5 安全性
所有网络请求通过 HTTPS。

敏感数据（如家长手机号）在数据库中加密存储（可选）。

使用云开发安全规则限制数据库访问权限（仅允许 openid 匹配的读写操作）。

5. 界面原型示意
5.1 首页（班级列表/工作台）
顶部显示教师头像、昵称。

下方显示班级卡片（可左右滑动），点击进入该班级主页。

班级主页底部 Tab：班级（学生列表）、成绩、评语、课表、考勤、通知。

点击“+”按钮快速添加学生/考试/评语等。

5.2 成绩录入页
选择考试，显示学生列表。

每行有学生姓名、分数输入框、缺考复选框。

底部固定栏显示当前统计（平均分、最高分等）。

右上角菜单：导出、完成。

5.3 评语生成页
上部分为模板选择区，按分类折叠。

下部分为学生列表，点击学生进入评语编辑。

评语编辑页：上方显示模板选择按钮，中间显示生成框，下方为手动编辑区，底部保存按钮。

5.4 课表页
周视图，格子内显示课程名称（若多节课可缩小字体）。

长按某格子进入编辑模式。

5.5 通知页
顶部为编辑表单（标题、正文、落款）。

中部为样式预览。

底部按钮“生成图片”，跳转预览页。

7. 开发建议
快速迭代：采用微信云开发，无需自建服务器，快速实现数据操作。先开发核心功能（班级、学生、成绩、评语），上线后收集反馈再迭代。

云函数：将复杂操作（如批量导入学生、成绩统计分析）封装为云函数，避免前端多次调用数据库。

数据库权限：使用“仅创建者可读写”的安全规则，确保数据隔离。

存储管理：生成的通知图片及时清理，避免存储空间浪费。

用户体验：成绩录入时支持键盘自动跳转下一格，类似Excel体验；评语生成建议加入快捷短语推荐。

推广：完成基础功能后，可在教师社群、小红书、知乎等平台分享，提供免费使用，吸引种子用户。
