# Teachers Aide - 云开发配置指南

## 1. 云环境配置

### 1.1 前提条件
- 已安装微信开发者工具（最新版本）
- 已注册微信小程序账号
- 已创建小程序项目

### 1.2 关联云环境
1. **打开微信开发者工具**，导入 `Teachers Aide` 项目
2. **点击左侧导航栏的「云开发」按钮**
3. **如果还没有云环境**：
   - 点击「创建云环境」
   - 输入环境名称（建议使用 `cloud1-7ggm9u41ff52487f`）
   - 选择「按量付费」
   - 点击「确定」创建
4. **如果已有云环境**：
   - 点击「设置」按钮
   - 在「环境设置」中，将「环境ID」设置为 `cloud1-7ggm9u41ff52487f`
   - 点击「确定」保存

### 1.3 配置项目文件
1. **打开 `miniprogram/app.js` 文件**
2. **确认云环境ID配置正确**：
   ```javascript
   wx.cloud.init({
     env: 'cloud1-7ggm9u41ff52487f',
     traceUser: true
   })
   ```

## 2. 云函数部署

### 2.1 云函数列表
项目包含以下云函数：
- `login`：用户登录和用户信息管理
- `openapi`：微信开放API调用
- `notice`：通知管理
- `schedule`：课程表管理

### 2.2 部署步骤
1. **在微信开发者工具中**，点击左侧导航栏的「云开发」按钮
2. **点击「云函数」标签页**
3. **点击「上传并部署」按钮**
4. **选择要部署的云函数**（可以选择全部）
5. **点击「部署」**，等待部署完成

### 2.3 云函数依赖安装
1. **在微信开发者工具中**，右键点击云函数目录（如 `cloud/login`）
2. **选择「在终端中打开」**
3. **运行命令**：
   ```bash
   npm install
   ```
4. **等待依赖安装完成**后，重新部署云函数

## 3. 数据库配置

### 3.1 需要创建的集合
项目需要创建以下数据库集合：
- `users`：用户信息
- `notices`：通知信息
- `schedule`：课程表信息

### 3.2 创建集合步骤
1. **在微信开发者工具中**，点击左侧导航栏的「云开发」按钮
2. **点击「数据库」标签页**
3. **点击「+ 集合」按钮**
4. **输入集合名称**（如 `users`）
5. **点击「确定」**创建集合
6. **重复步骤3-5**，创建其他集合

### 3.3 集合字段说明

#### users 集合
| 字段名 | 类型 | 描述 |
|-------|------|------|
| _id | String | 文档ID |
| openid | String | 用户openid |
| nickName | String | 用户昵称 |
| avatarUrl | String | 用户头像URL |
| gender | Number | 用户性别（0:未知, 1:男, 2:女） |
| province | String | 用户省份 |
| city | String | 用户城市 |
| country | String | 用户国家 |
| createTime | Date | 创建时间 |
| lastLoginTime | Date | 最后登录时间 |

#### notices 集合
| 字段名 | 类型 | 描述 |
|-------|------|------|
| _id | String | 文档ID |
| openid | String | 发布者openid |
| title | String | 通知标题 |
| content | String | 通知内容 |
| publishTime | Date | 发布时间 |
| createTime | Date | 创建时间 |
| isDeleted | Boolean | 是否删除 |

#### schedule 集合
| 字段名 | 类型 | 描述 |
|-------|------|------|
| _id | String | 文档ID |
| openid | String | 创建者openid |
| day | Number | 星期几（1-7） |
| timeSlot | Number | 节次（1-8） |
| courseName | String | 课程名称 |
| location | String | 上课地点 |
| createTime | Date | 创建时间 |
| isDeleted | Boolean | 是否删除 |

## 4. 云函数测试

### 4.1 测试方法
1. **在微信开发者工具中**，点击左侧导航栏的「云开发」按钮
2. **点击「云函数」标签页**
3. **选择要测试的云函数**（如 `login`）
4. **点击「测试」按钮**
5. **输入测试参数**（如 `{"code": "test_code"}`）
6. **点击「运行」**查看测试结果

### 4.2 测试用例

#### login 云函数
- **测试场景1**：用户登录
  - 参数：`{"code": "test_code"}`
  - 预期结果：返回用户信息

- **测试场景2**：更新用户信息
  - 参数：`{"action": "updateUser", "userInfo": {"openid": "test_openid", "nickName": "张老师", "avatarUrl": ""}}`
  - 预期结果：返回成功状态

#### notice 云函数
- **测试场景**：获取通知列表
  - 参数：`{"action": "list", "openid": "test_openid"}`
  - 预期结果：返回通知列表

#### schedule 云函数
- **测试场景**：获取课程表
  - 参数：`{"action": "list", "openid": "test_openid"}`
  - 预期结果：返回课程表数据

## 5. 常见问题及解决方案

### 5.1 云函数部署失败
- **问题**：部署云函数时提示「依赖安装失败」
- **解决方案**：
  1. 右键点击云函数目录，选择「在终端中打开」
  2. 运行 `npm install` 安装依赖
  3. 重新部署云函数

### 5.2 数据库访问失败
- **问题**：调用云函数时提示「数据库访问失败」
- **解决方案**：
  1. 检查云环境ID是否正确
  2. 检查数据库集合是否已创建
  3. 检查云函数的数据库操作权限

### 5.3 云函数调用失败
- **问题**：小程序调用云函数时提示「调用失败」
- **解决方案**：
  1. 检查云函数是否已部署
  2. 检查云环境ID是否正确
  3. 检查网络连接是否正常

### 5.4 权限问题
- **问题**：云函数调用时提示「权限不足」
- **解决方案**：
  1. 在云开发控制台中，点击「设置」
  2. 点击「权限设置」
  3. 确保云函数有相应的数据库操作权限

## 6. 最佳实践

### 6.1 开发建议
- **使用本地调试**：在开发过程中，可以使用本地模拟数据进行调试
- **定期部署**：修改云函数后，及时部署到云端
- **备份数据**：定期备份数据库数据

### 6.2 性能优化
- **减少云函数调用**：合并多个操作到一个云函数中
- **使用缓存**：对于频繁访问的数据，使用缓存
- **优化数据库查询**：使用索引优化查询性能

### 6.3 安全建议
- **验证用户身份**：在云函数中验证用户openid
- **参数验证**：对传入云函数的参数进行验证
- **错误处理**：添加适当的错误处理，避免敏感信息泄露

通过以上步骤，您可以成功配置微信云开发环境，部署云函数，创建数据库集合，并测试云函数的运行情况。如果遇到问题，请参考「常见问题及解决方案」部分，或查看微信云开发官方文档。
