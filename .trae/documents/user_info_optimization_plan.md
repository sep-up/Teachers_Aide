# 用户信息优化计划

## 需求概述
1. 去掉 `openid = 'test_openid'` 硬编码，不使用任何写死的本地数据
2. 获取用户真实头像并显示在页面上
3. 新用户弹窗提示编辑名称，默认填入微信昵称，存入数据库
4. 获取用户信息改成弹窗授权，授权完毕关闭

## 实施步骤

### 步骤1: 修改 login 云函数
- **文件**: `miniprogram/cloud/login/index.js`
- **操作**:
  - 移除第70行的 `openid = 'test_openid'` 备用方案
  - 当获取 openid 失败时，直接返回错误而不是使用备用 openid
  - 当查询用户不存在时，使用微信返回的 session_key 或其他标识创建新用户（如果微信返回了）

### 步骤2: 修改 openapi 云函数配置
- **文件**: `miniprogram/cloud/openapi/index.js`
- **操作**:
  - 使用微信开发者工具的云开发控制台配置环境变量 APP_ID 和 APP_SECRET
  - 或者在云函数配置中添加环境变量

### 步骤3: 修改首页显示用户信息
- **文件**: `miniprogram/pages/index/index.wxml`
- **操作**:
  - 修改头像显示逻辑，当有真实头像时显示，无头像时显示默认头像占位符
  - 确保头像正确显示

### 步骤4: 添加新用户编辑弹窗
- **文件**: `miniprogram/pages/index/index.js`
- **操作**:
  - 在 onLoad 中判断是否为新用户（根据 userInfo 是否有 nickName 或是否为空）
  - 如果是新用户，弹窗让用户编辑名称
  - 弹窗默认填入从微信获取的昵称（如果有）

### 步骤5: 改进 getUserInfo 授权流程
- **文件**: `miniprogram/app.js`
- **操作**:
  - 修改 getUserInfo 方法，在获取用户信息后自动更新页面数据
  - 确保授权弹窗正确显示
  - 授权完成后关闭弹窗或刷新页面数据
- **文件**: `miniprogram/pages/index/index.js`
- **操作**:
  - 在 getUserInfo 成功后，更新页面上的用户信息显示
  - 刷新用户数据

### 步骤6: 修改登录云函数支持新用户创建时获取昵称
- **文件**: `miniprogram/cloud/login/index.js`
- **操作**:
  - 修改创建新用户的逻辑，支持传入 nickName 参数
  - 确保新用户创建时使用传入的昵称

## 预期效果
- 用户首次打开小程序，会弹出授权窗口请求获取用户信息
- 用户授权后，头像和昵称会显示在首页
- 如果用户拒绝授权，显示默认头像和"教师"名称
- 新用户可以编辑自己的名称
- 不再使用任何写死的测试数据
